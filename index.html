How it works

A simple flow: fund once, estimate, send, track.

01
Provision an enterprise account
You receive an enterprise top-up wallet address and an auth token for X-Enterprise-Auth.
02
Top up in TON
Fund the enterprise wallet on-chain. Your balance is tracked in TON units.
03
Configure excess address (optional)
Fetch excess_account and use it as response_destination for jetton transfers to reduce effective fees.
04
Estimate upfront payment
Call estimate with a signed external message BOC and wallet public key. Get user fees calculated in TON.
05
Send & track
Send the same payload with send, then poll message status by msg_id to get final_cost and transaction hashes.
